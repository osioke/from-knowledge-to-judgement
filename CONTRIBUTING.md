# Contributing to From Knowledge to Judgement

Thank you for taking the time to contribute.

## What You Can Contribute

### Framework Refinements (Most Valued)
The five Thinking Frameworks in Appendix B are explicitly designed as living documents. If you have applied a framework in a real professional context and found:
- A triggering question that is missing
- A question that consistently produces the wrong focus
- A context (industry, geography, team size, company stage) where the framework needs a caveat
- A better way to phrase an existing question

— then a contribution is welcome.

### Career Recipe Contributions
If you have a Recipe — a real, tested, executable skill unit from your professional practice — you can contribute it to the appendix. Recipes must follow the format in Appendix C and must be based on real execution, not theoretical description.

### Corrections
Factual errors, broken links, typographical mistakes: raise an issue or submit a pull request directly.

## How to Contribute

1. Fork this repository
2. Make your changes in a new branch: `git checkout -b your-contribution-name`
3. Commit with a clear message describing what you changed and why
4. Open a pull request with a brief description of the contribution

## Standards

- All framework contributions must be grounded in professional practice, not theoretical frameworks alone
- Recipes must follow the five-component format (Domain, Skill Name, Trigger, Action Sequence, Expected Outcome)
- Keep language precise and practical — this is not a place for abstraction
- Context notes are encouraged where a contribution applies specifically to a particular industry, geography, or team size
